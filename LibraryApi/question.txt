Case Study
A small library wants to maintain book and member details digitally. The librarian
should be able to add books, view books, add members, and view members using a
Web API.
You need to create a Web API using ASP.NET Core and EF Core with proper
layered architecture.

Required Layers
Create the project using the following layers:

Required APIs
Book APIs
1. Add Book
POST /api/books
Sample Request:
{
&quot;title&quot;: &quot;Clean Code&quot;,
&quot;author&quot;: &quot;Robert C. Martin&quot;,
&quot;isbn&quot;: &quot;9780132350884&quot;,
&quot;publishedYear&quot;: 2008,
&quot;availableCopies&quot;: 5
}

Business Rules
1. Book title should not be empty.
2. Author name should not be empty.
3. Available copies should be greater than or equal to 0.
4. Member full name should not be empty.
5. Email should not be empty.
6. Phone number should not be empty.
7. If a book or member is not found by ID, return 404 Not Found.

Student Tasks
1. Create an ASP.NET Core Web API project.
2. Install EF Core packages.
3. Create models for Book and Member.
4. Create LibraryDbContext.

5. Configure SQL Server connection string.
6. Create repository interfaces and classes.
7. Create service interfaces and classes.
8. Create API controllers.
9. Register dependencies in Program.cs.
10. Create migration and update database.
11. Test all APIs using Swagger or Postman.
